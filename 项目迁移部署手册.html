一、项目概述

1.1 项目简介

本项目是一个基于AI的应届生简历生成系统，主要功能包括：

  • 智能简历生成：根据用户输入的个人信息自动生成专业简历
  • PDF导出：生成格式规范的PDF简历文件
  • Web管理界面：提供友好的用户操作界面

1.2 技术架构

层级技术栈
前端React + TypeScript + Tailwind CSS + Vite
后端Node.js + Express + TypeScript
数据库PostgreSQL + Drizzle ORM
AI服务Kimi AI API
PDF生成Python + ReportLab

四、详细部署流程

4.1 准备工作

步骤 1：复制迁移包

将桌面上的迁移包文件复制到新设备：

  • 迁移包文件夹: idea_0.2_migration_20260429_1624
  • 压缩包: idea_0.2_migration_20260429_1624.zip
提示：建议将迁移包解压到非中文路径，例如 D:\projects\ 或 C:\workspace\

步骤 2：解压迁移包

# 使用 Windows 资源管理器右键解压，或使用命令行：
Expand-Archive -Path "idea_0.2_migration_20260429_1624.zip" -DestinationPath "."

4.2 安装 Node.js 依赖

步骤 1：进入源代码目录

cd 01-source-code

步骤 2：安装依赖

npm install
注意：如果安装过程中出现网络错误，可以尝试使用淘宝镜像：
npm config set registry https://registry.npmmirror.com

步骤 3：验证安装

npm list

4.3 安装 Python 依赖

步骤 1：进入 Python 服务目录

cd ..\02-python-service

步骤 2：创建虚拟环境（推荐）

python -m venv venv
venv\Scripts\activate

步骤 3：安装依赖

pip install -r requirements.txt

4.4 配置环境变量

步骤 1：复制环境变量模板

cd ..\01-source-code
copy .env.example .env

步骤 2：编辑 .env 文件

使用文本编辑器打开 .env 文件，填写以下配置：

# 数据库配置
DATABASE_URL=postgresql://username:password@localhost:5432/resume_db

# AI API 配置
KIMI_API_KEY=your_api_key_here

# 其他配置
PORT=3000
NODE_ENV=development

4.5 数据库配置

步骤 1：创建数据库

# 使用 psql 命令行或 pgAdmin 创建数据库
createdb resume_db

步骤 2：运行数据库迁移

npm run db:migrate

4.6 启动服务

步骤 1：启动后端服务

npm run server

步骤 2：启动前端开发服务器（新终端）

npm run dev

步骤 3：访问应用

打开浏览器访问：http://localhost:5173

五、常见问题及解决方案

问题 1：npm install 失败

症状：安装过程中出现网络错误或权限错误

解决方案：

  1. 检查网络连接，尝试使用淘宝镜像：
    npm config set registry https://registry.npmmirror.com
  2. 以管理员身份运行命令行
  3. 清除 npm 缓存后重试：
    npm cache clean --force

问题 2：数据库连接失败

症状：启动服务时提示数据库连接错误

解决方案：

  1. 检查 PostgreSQL 服务是否已启动
  2. 验证 .env 文件中的数据库连接字符串是否正确
  3. 确认数据库用户具有足够权限

问题 3：Python 依赖安装失败

症状：pip install 报错

解决方案：

  1. 确保 Python 版本符合要求（3.10+）
  2. 升级 pip：
    python -m pip install --upgrade pip
  3. 使用国内镜像：
    pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple

问题 4：端口被占用

症状：启动服务时提示端口已被占用

解决方案：

  1. 查找占用端口的进程：
    netstat -ano | findstr :3000
  2. 结束占用进程或修改 .env 文件中的 PORT 配置
