# AI 功能配置与测试指南

## 配置 AI API Key

### 方法1：通过管理后台配置（推荐）

1. 使用管理员账号登录（admin / admin123）
2. 进入「管理后台」→「系统设置」
3. 添加设置项：
   - **Key**: `kimi_api_key`
   - **Value**: 你的 Kimi API Key（以 `sk-` 开头）
4. 点击保存

### 方法2：通过环境变量配置

在 `.env` 文件中添加：
```bash
KIMI_API_KEY=sk-your-api-key-here
```

然后重启服务器。

### 获取 Kimi API Key

1. 访问 [Moonshot AI 开放平台](https://platform.moonshot.cn/)
2. 注册/登录账号
3. 进入「API Key 管理」创建新密钥
4. 复制密钥（格式：`sk-xxxxxxxx`）

## 技术实现说明

### AI 调用流程

1. **前端** → 调用 tRPC API（不传递 apiKey）
2. **后端路由** → 调用 `chatWithKimi()`（不传 apiKey 参数）
3. **kimi-client** → 自动从 `system_settings` 表读取 `kimi_api_key`
4. **Kimi API** → 返回 AI 响应
5. **后端** → 解析响应并返回给前端

### 优先级

API Key 获取优先级：
1. 用户传入的 apiKey（已移除，统一使用系统配置）
2. 系统设置中的 `kimi_api_key`
3. 环境变量 `KIMI_API_KEY`
4. 模拟模式（返回预设数据）

## 故障排查

### AI 功能返回模拟数据

**现象**：分析结果是预设的模板数据，不是根据内容生成的

**原因**：
- 未配置 API Key
- API Key 无效或已过期
- 网络问题无法访问 Kimi API

**解决**：
1. 检查系统设置中是否配置了有效的 `kimi_api_key`
2. 检查 API Key 是否以 `sk-` 开头
3. 查看后端日志确认错误信息

### API 调用失败

**检查日志**：
```bash
# 查看后端日志
npm run dev
```

常见错误：
- `401 Unauthorized`: API Key 无效
- `429 Too Many Requests`: 请求频率过高
- `500 Internal Error`: 后端代码错误
